## 1. Current Split Construction Method

### How splits are currently built

The current split is generated by `scripts/prepare_data.py` which calls `split_records()` from `src/rag3d/datasets/builder.py`.

**Flow**:
1. NR3D annotations loaded from `data/raw/referit3d/annotations/nr3d_annotations.json`
2. Records built via `build_records_nr3d_hf_with_scans()`
3. Records split via `split_records()` with random shuffle
4. Manifests written to `data/processed/*.jsonl`

### Where random shuffle happens

**File**: `src/rag3d/datasets/builder.py`
**Function**: `split_records()` (lines 287-304)

```python
def split_records(
    records: list[dict[str, Any]],
    train_ratio: float = 0.8,
    val_ratio: float = 0.1,
    seed: int = 42,
) -> tuple[list[dict[str, Any]], list[dict[str, Any]], list[dict[str, Any]]]:
    import random

    rng = random.Random(seed)
    idx = list(range(len(records)))
    rng.shuffle(idx)  # <-- Random shuffle at SAMPLE level, not scene level
    n = len(idx)
    n_tr = int(n * train_ratio)
    n_va = int(n * val_ratio)
    tr = [records[i] for i in idx[:n_tr]]
    va = [records[i] for i in idx[n_tr : n_tr + n_va]]
    te = [records[i] for i in idx[n_tr + n_va :]]
    return tr, va, te
```

**Problem**: The shuffle operates on **sample indices**, not scene IDs. This means:
- Samples from the same scene can appear in different splits
- No guarantee of scene disjointness
- Violates standard 3D grounding evaluation practice

## 3. Where Scene Overlap is Introduced

Scene overlap is introduced at the **random shuffle step** in `split_records()`:

1. All 1,544 records are collected
2. Records are shuffled randomly (by index, not scene)
3. First 80% go to train, next 10% to val, last 10% to test
4. Since samples from same scene are distributed across the shuffled list, they can end up in different splits

**Example**: Scene `scene0467_00` has 6 samples. After random shuffle, these 6 samples may be distributed as:
- 4 samples in train
- 1 sample in val
- 1 sample in test

This creates scene overlap.

## 4. Cleanest Place to Enforce Scene-Disjoint Splitting

### Option A: Modify `split_records()` in builder.py

Add a new function `split_records_scene_disjoint()` that:
1. Groups records by scene_id
2. Shuffles scene IDs (not samples)
3. Assigns all samples of a scene to the same split

**Pros**: Clean, reusable, follows existing patterns
**Cons**: Need to handle ratio preservation (scene counts vs sample counts)

### Option B: Add new mode to `prepare_data.py`

Add `--mode build-nr3d-geom-scene-disjoint` that:
1. Calls a new split function
2. Writes manifests with scene-disjoint guarantee

**Pros**: Explicit mode, clear separation from old splits
**Cons**: More modes to maintain

### Recommendation: Option A + B combined

1. Add `split_records_by_scene()` in `builder.py`
2. Add `--mode build-nr3d-geom-scene-disjoint` in `prepare_data.py`
3. Store scene lists for each split for validation

## 5. Scene-Level Splitting Algorithm

```python
def split_records_by_scene(
    records: list[dict[str, Any]],
    train_ratio: float = 0.8,
    val_ratio: float = 0.1,
    seed: int = 42,
) -> tuple[list[dict[str, Any]], list[dict[str, Any]], list[dict[str, Any]]]:
    import random
    
    # Group records by scene
    by_scene: dict[str, list[dict]] = {}
    for r in records:
        sid = r["scene_id"]
        by_scene.setdefault(sid, []).append(r)
    
    # Shuffle scene IDs
    scene_ids = list(by_scene.keys())
    rng = random.Random(seed)
    rng.shuffle(scene_ids)
    
    # Assign scenes to splits
    n_scenes = len(scene_ids)
    n_tr = int(n_scenes * train_ratio)
    n_va = int(n_scenes * val_ratio)
    
    train_scenes = set(scene_ids[:n_tr])
    val_scenes = set(scene_ids[n_tr : n_tr + n_va])
    test_scenes = set(scene_ids[n_tr + n_va :])
    
    # Assign all samples of each scene to its split
    train = [r for sid in train_scenes for r in by_scene[sid]]
    val = [r for sid in val_scenes for r in by_scene[sid]]
    test = [r for sid in test_scenes for r in by_scene[sid]]
    
    return train, val, test
```

**Note**: Sample ratios will differ from scene ratios due to varying samples per scene.
