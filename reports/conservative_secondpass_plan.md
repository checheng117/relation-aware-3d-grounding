# Conservative Second-Pass Plan

## Scope and objective

This phase is a narrow follow-up to `outputs/20260331_180150_minimal_coadaptation/`.

Primary question:

- can second-pass reranker adaptation preserve the intermediate co-adapted gain (`0.1154`) instead of falling back to `0.1090`?

Hard constraints:

- no scope widening to hybrid router
- no phase **D** work
- no major architecture redesign

## Minimal code changes planned

1. Extend `scripts/train_two_stage_rerank.py` with minimal conservative-tuning knobs:
   - `early_stop_patience` (optional)
   - `min_delta` for early-stop and best-update thresholds
   - fine-module parameter selection mode (full vs partial/head-oriented)
2. Add one new orchestrator:
   - `scripts/run_conservative_secondpass_phase.py`
   - runs mandatory variants
   - writes required JSON/CSV/MD/PNG artifacts
   - runs corrected combined evaluation including best conservative row
3. Keep all old directories untouched; write new timestamped output directory only.

## Exact experiment order

1. README concise stage update (done first).
